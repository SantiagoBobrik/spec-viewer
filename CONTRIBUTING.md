# Contributing to Spec Viewer

Thank you for your interest in contributing to Spec Viewer! We welcome contributions from everyone.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone the project** to your local machine.
3.  **Install dependencies**:
    *   Go 1.21+
    *   Make sure you can run the project using `make run` or `air`.

## Development Workflow

1.  Create a new branch for your feature or bug fix. Please use one of the following prefixes: `feature/`, `fix/`, `docs/`, `refactor/`. Example: `git checkout -b feature/my-new-feature`.
2.  Make your changes.
3.  Commit your changes with clear messages.

## Submitting a Pull Request

1.  Push your branch to your fork.
2.  Submit a Pull Request to the `main` branch.
3.  Describe your changes clearly in the PR description.

## Code Style

*   Follow standard Go formatting (`go fmt`).
*   Keep code clean and readable.

## Reporting Issues

If you find a bug or have a feature request, please open an issue using the provided templates.
